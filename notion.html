1. 프로젝트 소개 및 주제 선정 이유

1.1 프로젝트 소개 및 주제 선정

<리그 오브 레전드의 로고, 사진 출처: Riot Games>

리그오브레전드(League of Legends, 이하 LoL)는 5명이 한 팀이 돼 전투를 벌이는 게임으로 상대의 본진 기지인 넥서스를 파괴하면 승리를 거두는 게임입니다. 2008년 ‘라이엇 게임즈’가 개발하였고, 배급한 멀티플레이어 온라인 배틀 아레나(MOBA) 게임의 대표 주자로서 전 세계에서 가장 인기 있는 게임입니다.

<사진 출처: 롤드컵 전세계 4억 명이 봤다…침체한 e스포츠 한국서 부활, 중앙일보, 고봉준 기자>

2019년 리그 오브 레전드의 동시 접속자 수는 800만 명을 돌파하였으며, 2021년 리그 오브 레전드 월드 챔피언십이라고 불리는 ‘롤드컵’의 경우 결승전의 최대 시청자 수는 7400만 명으로 집계될 정도로 선풍적인 인기를 끌고 있습니다.

165개의 챔피언과 다채로운 아이템들을 가지고 다양한 전략과 팀워크가 요구되는 이 게임에서 승리 요인을 분석하는 것은 플레이어와 팀에게 중요한 인사이트를 제공할 것입니다. 본 프로젝트에서는 라이엇 API를 활용하여 리그오브레전드에서 사용자들 간의 순위를 결정하는 랭크 게임 데이터를 수집하고, 이를 기반으로 탐색적 데이터 분석(Exploratory Data Analysis, EDA)과 여러 개의 머신 러닝 기법을 시행하여 승리 요인을 도출하였습니다. 이를 통해 게임 플레이에 대한 이해를 높이고, 승리 확률을 높이는 전략적 요소들을 파악하고자 합니다.

2. 데이터 전처리 및 데이터 파악

2.1 ETL (Extraction, Transform, Load)

2.1.1 라이엇 Open API 데이터 구조

  • 이 프로젝트에서 사용된 라이엇에서 제공하는 API 데이터 구조는 다음과 같습니다:

  • 이 API에서 데이터를 수집하는 과정은 다음과 같이 진행됩니다:
    1. LEAGUE-V4 API
      • 목적: 천상계 플레이어의 summonerID를 가져옵니다.
      • 출력: summonerID 리스트.
    1. SUMMONER-V4 API
      • 입력: LEAGUE-V4에서 받은 summonerID.
      • 목적: 각 summonerID에 연결된 puuid를 가져옵니다.
      • 출력: puuid 리스트.
    1. MATCH-V5 API
      • 입력: SUMMONER-V4에서 받은 puuid.
      • 목적: puuid를 사용해 matchID를 가져옵니다.
      • 출력: matchID 리스트.
    1. MATCH-V5 API (상세 매치 데이터)
      • 입력: 이전 MATCH-V5 호출에서 얻은 matchID.
      • 목적: 각 matchID에 대한 상세 매치 데이터를 가져옵니다.
      • 출력: 플레이어 통계, 팀 성과, 게임 이벤트 등을 포함한 매치 세부 정보.

2.1.2 데이터 추출(Extraction)

  • 라이엇 오픈 API: 천상계에 속하는 챌린저, 그랜드마스터, 마스터 티어의 소환사들을 대상으로 합니다.
  • 데이터 선택: 각 랭크에서 소환사 정보를 가져올 때, 전체 소환사 데이터를 다루기에는 범위가 넓기 때문에 효율적인 분석을 위해 30명의 소환사를 랜덤 샘플링 합니다.

2.1.3 데이터 변환(Transformation)

  • 소환사 식별자 수집: 각 랜덤 샘플링된 소환사의 summonerId를 추출하고, 이를 사용하여 puuid와 account_id를 가져옵니다.
  • 데이터프레임 생성: 수집된 summonerId, puuid, account_id를 포함하는 데이터프레임을 생성합니다.
  • 매치 데이터 수집: 각 소환사의 puuid를 사용하여 최근 20개의 matchId를 추출합니다. 추출 과정에서 중복되는 매치 데이터는 제거합니다.
  • 매치 데이터 조직: 추출된 매치 ID를 50개 단위로 그룹화하여 딕셔너리에 저장합니다. 이렇게 하여 각 딕셔너리 키는 소환사를, 값은 해당 소환사의 매치 리스트를 담게 됩니다.
  • 데이터 병합: 생성된 딕셔너리를 순회하면서 각 매치의 상세 정보(info)와 참가자 정보(participants)를 API에서 추출하여 데이터프레임으로 병합합니다. 이 과정에서 각 매치의 다양한 게임 데이터가 통합되어 하나의 데이터프레임으로 구성됩니다. 그 후, 랭크 별로 수집된 데이터프레임을 한 번 더 병합합니다.

2.1.4 데이터 로드(Load)

  • CSV 파일 저장: 최종적으로 병합된 데이터프레임을 CSV 파일 형식으로 저장하여 분석을 위한 데이터셋을 완성합니다. 이 파일은 추후 분석 단계에서 직접 불러와 사용됩니다.

# 패턴을 찾아 대체하는 함수 정의
def combine_names(champion_name, names):
    # 패턴을 찾아서 이름을 결합하는 함수
    def replace_name(match):
        name = match.group()
        if name in names and names.index(name) % 2 == 0:  # 인덱스가 짝수일 때(첫 번째 단어)
            return name + names[names.index(name) + 1]
        else:
            return name
    # 정규 표현식을 사용하여 패턴을 찾고, 찾은 패턴을 대체함
    replaced_text = re.sub(r'\b(?:' + '|'.join(names) + r')\b', replace_name, champion_name)

    return replaced_text

3.2.3 Logistic Regression

로지스틱 회귀는 분류 문제, 특히 이진 분류 문제에 자주 사용되는 통계적 모델입니다. 이 모델은 결과를 0과 1, 또는 '승리'와 '패배'와 같은 두 가지 범주 중 하나로 분류하는 데 효과적입니다. 가장 기본적인 벤치마크 모델이기에, 복잡한 모델을 적용하기 전 먼저 적용해 보았습니다.

3.2.3.1 모델 학습

  • 설정: 로지스틱 회귀 모델을 설정하여 리그 오브 레전드 게임의 승리 또는 패배를 이진 분류합니다. 이 모델은 게임 데이터의 여러 피처를 나타내는 독립 변수를 사용하여 승리(1) 또는 패배(0)의 확률을 예측합니다.
  • 모델 학습: 로지스틱 회귀 모델을 훈련 데이터 세트를 사용하여 학습시킵니다.
  • 피처 영향 분석: 모델의 계수를 분석하여 각 변수가 승리 확률에 미치는 영향의 크기와 방향을 평가합니다. 이 정보는 게임 전략을 조정하거나 팀 성능을 개선하는 데 사용될 수 있습니다.
  • 성능 지표: 모델의 예측 성능은 정확도와 F1-Score를 통해 평가합니다. 이러한 지표는 모델이 얼마나 잘 예측하는지를 측정하는 데 사용됩니다.
    • 성능 지표를 계산하는 방법:

3.2.4 Random Forest

랜덤 포레스트는 다수의 결정 트리를 조합해 과적합을 줄이고 일반화 성능을 향상시키는 앙상블 방법입니다. 여러 결정 트리를 통해 예측 성능을 평균으로 내어 모델에 성능을 향상시키고, 피처의 중요도를 평가할 수 있어 결과 예측에 중요한 요인을 식별할 수 있어 이 모델을 선정하였습니다.

  • 모델 구축: 랜덤 포레스트 모델을 구축하여 리그 오브 레전드 게임 데이터를 사용한 승리 예측을 수행했습니다. 200개의 결정 트리를 통합하여 각 트리의 예측을 평균 냄으로써, 보다 안정적이고 정확한 예측을 도모하였습니다.
  • 모델 학습: 랜덤포레스트 모델을 훈련 데이터 세트를 사용하여 학습시킵니다.
  • 변수 선택과 과적합 방지: 랜덤 포레스트의 내장 기능을 활용하여 가장 영향력 있는 변수를 식별했습니다. 이는 모델이 중요한 피처에 집중할 수 있게 하며, 과적합을 방지합니다.

3.2.5 Light Gradient Boosting Model (LGBM)

Gradient Boosting 모델을 적용해 보기 위해 효율성이 좋은 Light Gradient Boosting Model(LGBM)을 사용했습니다. LGBM Model은 트리 기반의 앙상블 모델이면서 그라디언트 부스팅 모델을 더욱 발전시킨 히스토그램 그라디언트 부스팅 모델로, 일반 그라디언트 부스팅 모델에 비해 학습하는 시간이 적고 과적합에 민감하여 과적합 없이 10000건이 넘는 데이터 세트의 승패를 분류를 잘할 것 같다고 생각하여 적용해 보았습니다.

3.2.5.1 모델 학습

  • 설정: Light GBM 모델을 설정하여 리그 오브 레전드 게임의 승리 또는 패배를 이진 분류합니다. 이 모델은 게임 데이터의 여러 피처를 바타내는 독립 변수를 사용하여 승리(1) 또는 패배(0)의 확률을 예측합니다.
  • 모델 학습: Light GBM 모델을 GridSearchCV를 이용한 하이퍼 파라미터 튜닝 탐색과 교차 검증을 진행하여 훈련 데이터 세트를 학습시킵니다.
  • 피처 영향 분석: 모델의 계수를 분석하여 각 변수가 승리 확률에 미치는 영향의 크기와 방향을 평가합니다. 이 정보는 게임 전략을 조정하거나 팀 성능을 개선하는 데 사용될 수 있습니다.
  • 성능 지표: 모델의 예측 성능은 정확도와 F1-Score를 통해 평가합니다. 이러한 지표는 모델이 얼마나 잘 예측하는지를 측정하는 데 사용됩니다.

‘메타 챔피언 픽을 한 팀이 승리할 확률이 높을 것이다.’라는 가설을 세우고 EDA를 진행한 결과를 다음과 같이 요약할 수 있습니다.

  1. 메타 챔피언을 픽한 팀의 승률이 픽하지 않은 팀보다 1.1% 정도 더 높다.
  1. 메타 챔피언을 픽하는 게임은 약 79%이며, 픽하지 않는 게임은 전체의 21%를 차지한다. 그뿐만 아니라, 천상계 게임에서는 메타 챔피언이 1개만 픽 되는 경우가 가장 많았고, 2개(26.6%), 0개(21.1%), 3개(8.7%), 4개(1.2%), 5개(0.1%) 순으로 나타났다.
  1. 메타 챔피언의 픽 수에 따라 골드 수급량의 차이는 없으며 픽 유무에 따라 포탑에 입힌 피해량, 건물에 입힌 피해량, 상대방에게 입힌 피해량의 차이는 나지 않는다.
  1. 다만, 메타 챔피언을 픽한 팀의 경우 데미지 감소/방어량에 있어서 픽하지 않은 팀보다 높게 나타났다.
  1. 메타 챔피언들은 ‘bottom’, ‘utility’ 포지션에서 가장 많이 선택되었으며, 이는 ‘bottom’,’utility’ 포지션에서 메타 챔피언을 선택하는 것이 중요하다고 해석될 여지가 있다.
  1. 가장 많이 선택된 메타 챔피언은 ‘Nautilus’이며, 정통적인 원딜 챔피언과 방어력과 방어 기능을 제공해주는 튼튼한 탱커 계열의 챔피언이 주로 선택되었다.
  1. ‘bottom’,’utility’ 포지션의 챔피언들이 많이 선택되어 ‘bottom’,’utility’ 챔피언들을 픽하면 승률이 높을 것으로 생각하고 승률을 확인하였더니 ‘Ashe’,’Caitlyn’의 경우에만 승률이 55%를 넘고 다른 챔피언들은 패배율이 더 높거나 평균에 근사했다.
  1. ‘Ashe’의 경우 승률이 높은 이유로 강력한 CC기라고 생각될 수 있지만, ‘Caitlyn’의 경우 승률이 높은 정확한 원인을 해당 데이터에서 생각해 낼 수 없었다.

해당 EDA의 결과를 이용했을 때 시사할 수 있는 점은 메타 챔피언 픽 여부에 따라 승리에 영향을 줄 수 있는 약한 변수로 생각될 수 있지만, 승리에 강력한 영향을 줄 수 있는 변수로는 생각되지 않을 수도 있다고 사료 됩니다. 메타 챔피언의 픽 여부에 따라 ‘골드 획득량’, ‘입힌 피해량’ 등을 같이 시각화해 본 결과 변수에 확실히 영향을 준다고 보여지지 않기 때문입니다.

메타 챔피언을 픽한다면, ‘bottom’ 포지션의 픽을 하는 것이 중요할 수 있으며, 방어 능력치가 높은 챔피언이거나 방어적인 특성을 제공하는 챔피언을 픽하는 것이 유행하기 하고 있기 때문에 방어 특성을 제공하는 챔피언을 픽하거나 방어적인 챔피언을 약점으로 삼을 수 있는 카운터 챔피언을 픽하여 상대를 공략하는 것이 게임을 승리로 이끌 수 있습니다. 혹은 방어적인 특성을 지닌 챔피언과 상대적으로 ‘Caitlyn’이나 CC기 기반을 활용한 강력한 라인전을 펼치는 ‘Ashe’ 결합하여 ‘bottom’ 픽 조합을 구성하여 승리를 도모해보는 것도 승리를 위한 좋은 방법으로 생각됩니다.

4.1.2 ‘오브젝트를 많이 획득한 팀이 승리할 확률이 높을 것이다.’의 대한 결론

  • EDA에서 바론, 드래곤, 장로드래곤을 획득하였을 때 팀의 승률이 높아진다는 것을 알 수 있었습니다.
  • 게임 내에서 오브젝트를 획득하기 유리한 정글 챔피언을 알아보았을 때 상위 10명의 챔피언 중 'Ekko', 'RekSai', 'Vi', 'Khazix', 'Nunu', 'Talon’이 있고 이는 정글 챔피언 승률 상위 10명의 데이터와 일치하였습니다.
  • 그러므로 해당 EDA를 통해 승률을 높이기 위한 전략으로 오브젝트 획득률이 높은 챔피언을 통해 오브젝트 획득 횟수를 늘리는 것이 좋다고 생각됩니다.

4.1.3 ‘시야 점수가 높을수록 승리할 확률이 높을 것이다.’ 의 대한 결론

  • 상관관계 분석: 제공된 상관관계 매트릭스에서 wardsPlaced, wardsKilled, visionScore, visionWardsBoughtInGame 등의 지표들과 승리('win') 간의 상관계수는 모두 0.1 미만으로 나타났습니다. 이는 이러한 지표들이 승리와 직접적인 강한 연관성이 없음을 나타냅니다.
  • 가설 검토: 천상계에서는 팀의 와드 사용 및 시야 확보 관리가 일관적으로 잘 이루어지는 경향이 있습니다. 따라서, 와드 관련 지표가 승리와 큰 차이를 보이지 않는 것으로 보입니다.

4.2 ML Modeling에 대한 결론 및 전략

4.2.1 모델 평가

  • 리그 오브 레전드 천상계 매치 데이터를 사용하여 로지스틱 회귀, 랜덤 포레스트, 그리고 LGBM 모델을 적용한 결과, 랜덤 포레스트와 LGBM 모델이 F1-score 0.9로 가장 높은 예측 성능을 보였습니다. 실제 게임 데이터에 적용했을 때, 두 모델 모두 약 0.88의 예측 정확도를 달성하였습니다.
  • 또한 두 모델의 피처 중요도는 다음과 같이 나타났습니다:
    • 랜덤 포레스트 주요 피처 순위:
      • bountyLevel
      • totalTimeSpentDead
      • inhibitorTakedowns
      • damageDealtToObjectives
      • totalDamageDealt
      • goldEarned
    • LGBM 주요 피처 순위:
      • totalDamageDealt
      • damageDealtToObjectives
      • totalTimeSpentDead
      • goldEarned
      • totalHeal
      • champLevel

4.2.1 결론 및 전략

  • 모델들이 공통으로 중요하게 평가한 피처는 다음과 같습니다:
    • totalTimeSpentDead
    • damageDealtToObjectives
    • totalDamageDealt
    • goldEarned
  • 이 피처들은 승리 예측에 있어 중요한 역할을 하는 것으로 나타났습니다. 특히, totalTimeSpentDead과 damageDealtToObjectives는 게임의 진행에 중대한 영향을 미치는 요소로, 플레이어의 생존 시간과 목표물에 대한 피해가 게임 결과에 직접적인 영향을 미치는 것을 보여줍니다.
  • 다른 한편으로, bountyLevel, champLevel, totalHeal과 같은 피처는 더 민감하게 반응하는 변수로, 특정 상황에서 게임의 향방을 결정짓는 요소로 작용할 수 있습니다.
  • 천상계 유저를 위한 전략적 권장 사항:
    • 효율적인 오브젝트 관리
      • 목표물에 대한 피해 증대: damageDealtToObjectives는 승리 예측에 중요한 요소로 나타났습니다. 플레이어는 드래곤, 바론, 터렛 등의 게임 오브젝트에 더 큰 피해를 주도록 노력해야 합니다.
      • 시간 관리와 자원 배분: 플레이어는 목표물을 효과적으로 활용하여 자원을 최적화하고, 목표물을 빠르게 처치함으로써 경기에서의 우위를 점할 수 있습니다.
    • 생존성 강화
      • 사망 시간 최소화: totalTimeSpentDead이 승리에 부정적인 영향을 미치므로, 플레이어는 위치 선정과 전투 참여 결정에 있어 보다 신중을 기해야 합니다.
      • 안전한 플레이 유도: 과도한 리스크는 피하고, 팀과의 긴밀한 협력을 통해 생존성을 높이는 전략을 채택해야 합니다. 죽지 않을수록, 이기는 확률이 높아집니다.
    • 자원 획득의 최적화
      • 골드 획득에 집중: 'goldEarned'는 승리에 중요한 요인입니다. 플레이어는 미니언 & 정글 몬스터 처치, 그리고 킬/어시스트를 통해 가능한 한 많은 골드를 획득하도록 노력해야 합니다.
      • 아이템 구매와 업그레이드: 획득한 골드를 잘 이용하여 게임 상황에 맞는 아이템을 신속하게 구매하고 업그레이드함으로써 전투력을 강화해야 합니다.

4.3 기대효과 및 한계점(Limitation)

4.3.1 기대효과

  1. 효과적인 자원 관리
  • 골드와 시간의 최적화: 천상계에서 중요하게 평가된 'goldEarned'와 'totalTimeSpentDead'는 게임 내 자원 관리의 중요성을 강조합니다. 이를 통해 유저들이 효율적인 골드 사용법과 사망을 피하는 전략을 배우는 것을 지향하면 이는 게임 전반적인 성능 향상으로 이어질 것입니다.

2. 목표 지향적 플레이

  • 목표물 중심의 게임 운영: 천상계 데이터에서 'damageDealtToObjectives'의 중요성이 부각됩니다. 이를 바탕으로 한 전략적 플레이를 장려함으로써 전체 게임 흐름을 효과적으로 제어하는 방법을 배울 수 있습니다.

3. 메타 게임 이해 및 적응

  • 최신 게임 트렌드 분석: 천상계의 데이터를 분석함으로써 최신 메타와 챔피언 트렌드를 신속하게 파악하고, 이를 일반 유저들에게 적절한 형태로 전달합니다. 유저들은 현재 게임 환경에 가장 적합한 챔피언 선택과 전략을 배울 수 있습니다.

4.3.2 한계점

  1. 데이터의 한계
    • 데이터 샘플링과 선택편향: 랜덤 샘플링과 특정 등급의 플레이어만을 대상으로 한 분석은 전체 플레이어 인구를 완벽하게 대표하지 못할 수 있습니다. 이는 선택편향을 초래하며, 특정 유형의 플레이어만이 포함되어 일반화 가능성을 제한할 수 있습니다.
    • 데이터양과 질: 데이터의 양이 충분하지 않거나, 수집 과정에서의 오류, 누락된 값 등은 결과의 정확도에 영향을 줄 수 있습니다.
  1. 모델 관련 한계
    • 모델의 일반화: 모델이 특정 데이터 세트에 과적합 될 가능성이 있으며, 다른 조건이나 설정에서는 같은 성능을 보장하지 못할 수 있습니다.
    • 피처 중요도의 해석과 측정편향: 랜덤 포레스트와 LGBM에서 도출된 피처 중요도는 각 모델의 내부 계산 로직에 따라 달라집니다. 이는 측정 방법의 차이에 따른 측정편향을 발생시킬 수 있으며, 다른 모델과의 비교에서 피처의 영향을 일관되게 해석하는 데 한계가 있을 수 있습니다.
  1. 분석 방법의 한계
    • 통계적 검증과 변수 처리: 상관관계 분석이 인과 관계를 설명하지 못하고, 변수 선택 및 처리 과정에서 주관적 판단이 개입될 수 있습니다. 이는 분석 결과에 편향을 일으킬 수 있습니다.
  1. 외부 요인의 무시
    • 팀 상호작용과 전략: 피처 외에도 팀의 협력, 전략 실행 능력 등이 게임 결과에 중대한 영향을 미치지만, 이러한 요인은 종종 분석에서 누락됩니다.
    • 게임 패치와 업데이트: 리그 오브 레전드의 정기적인 게임 패치와 업데이트는 게임 메타와 전략에 큰 영향을 미치며, 이러한 변화는 시간에 따라 분석 결과에 영향을 줄 수 있습니다.
